서버에 올리지 않은 로컬의 HTML 파일을 Chrome 브라우저를 통해 Ajax 사용 시, CORS 에러가 발생한다. 
이를 해결하기 앞서, CORS(Cross Origin Resource Sharing)와 SOP(Same-Origin-Policy)에 대해 알 필요가 있다. 

SOP란 웹 애플리케이션 보안 모델에서 중요한 개념 중 하나인 정책이다. 
이 정책에 의해 스크립트(XMLHttpRequest)로 다른 웹 페이지로 접근할 때는, 출처(프로토콜, 호스트명, 포트)가 같아야만 한다. 
즉, 웹 페이지의 스크립트는 그 페이지와 같은 서버에 있는 주소로만 Ajax 요청을 할 수 있다. 
하지만 웹의 진화로 인해 여러 도메인을 사용하는 대규모 프로젝트가 늘어나고, REST API 등을 이용한 외부 호출이 많아지며 SOP 정책은 효율이 떨어지게 되었다. 

CORS란 웹 브라우저에서 외부 도메인 서버와 통신하기 위한 방식을 표준화한 스펙이다. 
CORS 방식은 요청을 받은 웹 서버가 허용할 경우, 다른 도메인의 웹 페이지 스크립트에서도 자원을 주고받을 수 있다. 
따라서, 교차 출처 자원 공유 방식인 CORS를 사용한다면 외부 도메인 요청 가능 여부를 확인하는 동작(SOP 정책)을 무시하게 할 수 있다. 

웹 브라우저(Chrome)에서 커맨드 라인 옵션 조작을 통한 SOP 무효화 
	1. Chrome 브라우저 바로 가기 생성 
	2. 우 클릭 > 속성(R) > 바로 가기 탭 > 대상(T): > 텍스트 필드 수정 
	3. 텍스트에 ' --disable-web-security --user-data-dir="바로 가기 경로"' 추가 
	4. 실행 후, 좌측 하단의 '지원되지 않는 명령 줄 플래그(--disable-web-security)를 사용 중이므로 안정성과 보안에 문제가 발생합니다.' 텍스트 확인 

* 첫 실행 시, 디렉터리 및 파일이 여러 개 생성됨으로 새 폴더 생성 후, 작업하는 것을 추천 
* Ajax로 로컬 파일을 불러오는 부분에서 인코딩 문제 발생 시, 해당 파일의 인코딩 확인 필요
